### Понятие информационной системы
Информационная система (ИС) — это совокупность взаимосвязанных компонентов, работающих вместе для сбора, хранения, обработки и распределения информации. Основная цель ИС — поддержка принятия решений и управление операциями в организациях. ИС включает в себя аппаратное и программное обеспечение, базы данных, сетевые ресурсы и процедуры, обеспечивающие выполнение задач.

### Доступность
Доступность (availability) в контексте информационных систем — это мера, показывающая, насколько система готова к использованию в любой момент времени. Доступность определяется как соотношение времени, когда система доступна для пользователей, к общему времени. Высокая доступность критически важна для многих приложений и достигается через резервирование компонентов, отказоустойчивость и регулярное обслуживание.

### Понятие иерархии
Иерархия — это система упорядочивания элементов, в которой элементы на более высоких уровнях контролируют или управляют элементами на более низких уровнях. В контексте баз данных и ИС иерархия может относиться к структуре данных, где данные организованы в виде дерева (например, иерархическая модель данных), или к организационной структуре, где различные уровни управляют друг другом.

### Практическое задание

Создадим базу данных «Поликлиника» с тремя сущностями: пациенты, врачи и услуги. Затем напишем SQL-запрос, который выводит три записи пациентов, осматривающихся у офтальмолога.

```sql
-- Создание базы данных
CREATE DATABASE Poliklinika;
USE Poliklinika;

-- Создание таблицы пациентов
CREATE TABLE Pacienty (
    id INT PRIMARY KEY AUTO_INCREMENT,
    imya VARCHAR(255),
    familiya VARCHAR(255),
    vozrast INT,
    kontakt_info VARCHAR(255)
);

-- Создание таблицы врачей
CREATE TABLE Vrachi (
    id INT PRIMARY KEY AUTO_INCREMENT,
    imya VARCHAR(255),
    familiya VARCHAR(255),
    spetsializatsiya VARCHAR(255),
    kontakt_info VARCHAR(255)
);

-- Создание таблицы услуг
CREATE TABLE Uslugi (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nazvanie VARCHAR(255),
    opisanie TEXT,
    vrach_id INT,
    pacient_id INT,
    FOREIGN KEY (vrach_id) REFERENCES Vrachi(id),
    FOREIGN KEY (pacient_id) REFERENCES Pacienty(id)
);

-- Запрос на выборку трех записей пациентов, осматривающихся у офтальмолога
SELECT Pacienty.imya, Pacienty.familiya, Vrachi.spetsializatsiya
FROM Uslugi
JOIN Pacienty ON Uslugi.pacient_id = Pacienty.id
JOIN Vrachi ON Uslugi.vrach_id = Vrachi.id
WHERE Vrachi.spetsializatsiya = 'Офтальмолог'
LIMIT 3;
```

В этом запросе мы используем операторы `JOIN` для объединения таблиц пациентов, врачей и услуг по соответствующим внешним ключам. Условие `WHERE Vrachi.spetsializatsiya = 'Офтальмолог'` фильтрует услуги, предоставляемые офтальмологами, а оператор `LIMIT 3` ограничивает результат тремя записями.

Этот подход демонстрирует основные навыки работы с базами данных, включая создание таблиц, установление связей между ними и выполнение сложных запросов.
